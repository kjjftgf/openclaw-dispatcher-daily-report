## 📊 日报六板块结构

### Sheet 1: 全天达成
7列：排名 / 骑手 / 完单量 / 实在线 / 是否达成 / 是否在线 / 差距

基线：≥6h 且 ≥18单
信息行：站点人数 → 排班人数 → 应排人数 → 达成人数 → 未达成人数

### Sheet 2: 时段未达标 — 区分已结束/进行中/未开始
| 列 | 内容 |
|:--:|------|
| A | 时段名（9个时段）|
| B | 应排人数（排班表上的人数）|
| C | 实排人数（实际上线人数 = `present`）|
| D | 达标人数（完成时段要求的人数）|
| E | 未出勤（应排 − 实排）|
| F | 出勤不达标（实排 − 达标）|
| G | 出勤率（实排÷应排，对照监控表 paRate）|
| H | 达标率（达标÷应排，对照监控表 sqRate）|
| I | 天汇总应排出勤率（Σ出勤率×权重÷总权重）|

**出勤率判定阈值：85%**
**全天聚合达标率阈值：`ceil(应排人数×0.85)`**

各时段明细包含预测列（工作状态/配送中/预测）：
- **进行中时段预测**：用全天效率兜底（时段在线<1h 则参考全天效率 ×0.6）
- 预测显示「是」或「否」

### Sheet 3: 档位达成
各档位分布 + 接近升级骑手 + 低于基线名单

档位标准（取低原则）：

| 档位 | 在线 | 完单 |
|:----:|:----:|:----:|
| 标准5 | 8h | 53单 |
| 标准4 | 7h | 43单 |
| 标准3 | 6h | 35单 |
| 标准2 | 6h | 28单 |
| 标准1 | 6h | 18单 |

### Sheet 4: 重点跟进
冲三档（二档→三档，差≤2单或≤20min）+ 全部升档机会 + 汇总

### Sheet 5: 0-2档不够明细
展示 level 0/1/2 骑手到下一档的差距。
**9列：** 骑手 / 目标档位 / 有效完单 / 有效时长 / 配送中单量 / 是否在线 / 差单 / 差时 / 能否达成

- 动态计算到下一档的差距（差单、差时）
- 不含标三及以上骑手

### Sheet 6: 档位明细
展示 0~4 档每位骑手到下一档的差距（标五已到顶不展示）。
**10列：** 骑手 / 档位 / 实完单 / 有效时长 / 在线时长 / 配送中 / 差单 / 晋升档位 / 后续班次 / 达成

#### 格式细节（2026-07-07 确定版）：
- **无档位分隔头**：各档位直接连续排列，无 "标准4→标准5（8人）" 这类分隔行
- **无改派列**：K列已删除，共10列
- **各档位不同底色**：
  - 标准4（level 4）：蓝色系（BBDEFB）
  - 标准3（level 3）：绿色系（C6F6D5）
  - 标准2（level 2）：青色系（B2EBF2）
  - 标准1（level 1）：紫色系（E8D5FF）
  - 不足标准1（level 0）：粉色系（FFCCCC）
- **底色优先级**：差1-2浅黄高亮 > 改派橙（未达成时）> 达成绿 > 档位色斑马纹
- **差1-2单高亮**：浅黄(FFF9C4)整行底色 + 深橙(D84315)粗体差单数字
- **差单≥3**：普通红色(E53E3E)字体
- header 位于所有数据之前，仅一次

**档位 → 下一档目标定义：**

| 当前档位 | 下一档 | 目标单量 | 目标时长 |
|:--------:|:------:|:--------:|:--------:|
| 0（不足标1）| 标准1 | 18单 | 6h |
| 1（标准1）| 标准2 | 28单 | 6h |
| 2（标准2）| 标准3 | 35单 | 6h |
| 3（标准3）| 标准4 | 42单 | 7h |
| 4（标准4）| 标准5 | 53单 | 8h |

## 📈 7月 KPI 考核规则

### 结算门槛
- **全天有效出勤率** ≥ 90%
- **时段有效出勤率** ≥ 85%
- 低于门槛 → 不结算基础服务费

### 尖峰场景加减分（标2-标5占比）
- **分母 = 当天应排人数**
- **>70%目标** = 达标加1分（严重恶天加2分）
- **<30%警告** = 占比 < 30% → 扣1分（严重恶天扣2分）
- **30%-70%** = 不扣不加
- **严重恶天** = 34级天气标签占比 ≥ 50%（3级极恶劣+4级罕见恶劣）
- **月度累计上限** = ±10分
